In the world of project management, "Analyse Fonctionnelle" (Functional Analysis) is often seen as a professional engineering or industrial design step. However, for the **general public**, several web apps simplify these complex concepts—like "External Functional Analysis" (the Octopus diagram) or "Requirement Listing"—into user-friendly tools.

If you are looking for apps to define a project by its functions and needs, here are the best options for everyday users:

### 1. Visual Brainstorming & "Octopus" Diagrams
In functional analysis, you often need to see how your project interacts with its environment. These tools are perfect for creating **pieuvre (octopus) diagrams** or **functional trees**.
* **Miro / Mural:** These are digital whiteboards. They have specific templates for "Mind Mapping" or "Concept Mapping" that you can easily use to identify "Functions of Constraint" and "Main Functions."
* **MindMeister:** A dedicated mind-mapping tool. It’s excellent for "Functional Decomposition"—taking one big goal and breaking it down into the smaller sub-functions required to make it work.

### 2. Structured Requirement Apps (The "What" before the "How")
These apps help you list **user needs** and **technical requirements** without getting bogged down in engineering software.
* **Notion:** While known for notes, Notion is powerful for project definition. You can create a "Requirements Database" where each property represents a functional criteria (e.g., "Priority," "Cost," "Target User"). It’s the best "all-in-one" for the general public to document a project scope.
* **Coda:** Similar to Notion but more "math-heavy." It allows you to build custom tables that act like mini-apps. You can use it to create a weighted scoring system for your project's functions (Value Analysis).

### 3. User-Centric Definition (User Stories)
If your "functional analysis" is for a digital product or a service, these tools help you define the project based on **user behavior**.
* **Trello:** Use the "User Story Mapping" method. Each list is a step in the user’s journey, and each card is a "function" or feature needed at that step.
* **Draft.io:** A French-born tool (great if you prefer the "Analyse Fonctionnelle" terminology) that combines whiteboarding with structured project management. It’s very visual and great for mapping out flows.

### 4. Specialized "Lightweight" Scoping Tools
* **Delibr:** This is a niche tool specifically for "Product Documents." It helps you outline the problem, the solution, and the functional requirements in a structured way before you start building.
* **Gleek.io:** A unique tool where you **type** descriptions and it automatically generates diagrams (like flowcharts or class diagrams). It's great if you want a professional look without the manual "dragging and dropping."

### Comparison Summary: Which one should you pick?

| If you want to... | Use this App |
| :--- | :--- |
| **Visualize interactions** (Octopus/Pieuvre) | **Miro** |
| **List & categorize requirements** | **Notion** |

| **Create quick technical diagrams** | **Gleek.io** |
