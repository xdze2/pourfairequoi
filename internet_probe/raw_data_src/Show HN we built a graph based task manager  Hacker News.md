First off, why another task manager / todo app?

Well, because the current ones kinda underperform. They work great for a small amount of tasks, but as soon as you add more they become chaotic! You quickly lose the overview, sub-lists get messy and finding your todo’s between your teammates takes ages.

Moreover, current solutions limit your actions! They only allow you a linear execution order of tasks (top-to-bottom). And modeling task dependencies in 1 dimension just doesn’t work. If you don’t believe us, try converting the example on our webpage (link below) into a traditional todo list!

Frustrated with the above, my friend and I decided to build Masterplan! A graph based task manager where tasks are nodes in a graph and where you can link them to create dependencies. Getting an overview of the _whole_ project is now just a zoom-out away. And dependencies are immediately visible. No more deep nesting or a million sub-lists!

Plus, collaboration in a team is super easy. Using state-of-the-art blockchain tech we made it so that multiple people can edit the same task graph at the same time with a latency of just 15 min! Just kidding, we did that without blockchains or latency ;) Moreover, tasks can be delegated to others with ease and team members have a special view which compacts the graph so that only tasks delegated to them are visible.

We're not yet ready to launch, but wanted to share our work anyway.
